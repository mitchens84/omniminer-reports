## ⚡ BOTTOM LINE

We're witnessing the emergence of an "agent fork" of the internet where AI agents will interact with structured, programmable interfaces rather than visual web interfaces, potentially creating new trillion-dollar companies that couldn't exist on the human web.

---

## 📝 THESIS

The internet is undergoing a fundamental architectural shift similar to the 2008 mobile fork—from visual interfaces designed for humans to structured, programmable interfaces optimised for AI agents. This "agent fork" will be defined not by startups but by infrastructure giants (Coinbase, Stripe, Cloudflare, Google, OpenAI, Visa, PayPal) who can turn their design decisions into de facto web standards for agent-to-agent commerce and interaction.

---

## 💡 KEY INSIGHTS

1. **The agent web requires structured, transactional interfaces, not visual ones** — AI agents need machine-readable APIs, programmable payment rails, and structured data rather than visual layouts designed for human consumption.[^1]

2. **Infrastructure giants, not startups, will define the agent web standards** — Companies with existing scale, distribution, and infrastructure (Coinbase, Stripe, Cloudflare, Google, OpenAI, Visa, PayPal) are positioned to make their design decisions into de facto standards.[^2] [✓]

3. **The mobile fork analogy reveals how interface primitives create new business categories** — Just as mobile's real-time location, always-on connectivity, and camera-first interaction enabled Uber, Instagram, WhatsApp, and Snap, the agent fork will create businesses that couldn't exist on the human web.[^3]

4. **Agent clients need different interface primitives than humans** — Agents require structured data access, programmable transactions, machine-readable contracts, and autonomous decision-making capabilities rather than information presentation.[^4]
